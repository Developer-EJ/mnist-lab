## 1. 실험 목적

MNIST 10-class 분류를 **NumPy만으로 구현한 신경망**으로 수행하고, 테스트 정확도와 학습 과정을 보고합니다.

---

## 2. 모델 구조


| 구분      | 내용                                                               |
| ------- | ---------------------------------------------------------------- |
| **입력**  | 784 (28×28 픽셀, 0~1 정규화)                                          |
| **은닉층** | Affine → BatchNorm → ReLU → Dropout 순으로 구성 |
| **출력**  | Affine(→10) + Softmax                                            |


**모델 구조 : 2층 은닉**  
입력 784 → Affine(512) → BatchNorm → ReLU → Dropout → Affine(256) → BatchNorm → ReLU → Dropout → Affine(10) → Softmax

## 6. 회고

이번 실험에서는 MNIST 손글씨 숫자 분류를 NumPy만으로 직접 구현한 신경망을 사용하여 학습해 보았다. 단순히 라이브러리를 사용하는 것이 아니라, Affine 계층, BatchNorm, ReLU, Dropout, Softmax, Adam Optimizer 등을 직접 구현하면서 신경망의 학습 과정이 어떤 방식으로 이루어지는지 이해할 수 있었다.
\
\
6.1 손실 수렴 여부

기본 설정에서는 학습이 안정적으로 진행되었고, 최종 테스트 정확도는 약 **98.45%**로 비교적 높은 성능을 얻을 수 있었다. Train Loss와 Test Loss가 전반적으로 감소하는 모습을 보였기 때문에 모델이 MNIST 데이터의 패턴을 잘 학습했다고 볼 수 있다.

특히 Adam Optimizer와 적절한 learning rate인 0.001을 사용했을 때 학습이 안정적으로 수렴하였다. 반면 learning rate가 너무 큰 경우에는 손실이 제대로 줄어들지 않았고, 너무 작은 경우에는 학습 속도가 느려져 충분히 수렴하지 못하는 모습을 확인할 수 있었다.

---
6.2 Batch Size 실험 회고

Batch Size 비교 실험에서는 다음과 같은 차이를 확인할 수 있었다.
| Batch Size | 특징 |
|---:|---|
| 10 | 가장 높은 정확도인 **98.49%**를 기록했지만 학습 시간이 **1790초**로 매우 길었다. |
| 128 | 정확도 **98.38%**로 batch size 10과 큰 차이가 없으면서 학습 시간이 훨씬 짧았다. |
| 60000 | 전체 데이터를 한 번에 학습하여 시간은 짧았지만 정확도가 **92.50%**로 크게 낮아졌다. |

이를 통해 batch size가 너무 작으면 학습은 세밀하게 이루어질 수 있지만 시간이 오래 걸리고, 너무 크면 업데이트 횟수가 줄어들어 학습이 충분히 이루어지지 않을 수 있다는 것을 확인했다.

따라서 이번 실험에서는 batch_size=128이 성능과 학습 시간의 균형이 가장 좋은 설정이라고 판단할 수 있었다.

---

6.3 Learning Rate 실험 회고

Learning Rate 실험에서는 학습률이 모델 성능에 매우 큰 영향을 준다는 것을 확인했다.

| Learning Rate | 결과 해석 |
|---:|---|
| 100 | 손실이 매우 크고 정확도도 **9.58%**로 거의 학습되지 않았다. |
| 0.1 | 어느 정도 학습은 되었지만 최적 성능에는 미치지 못했다. |
| 0.001 | 가장 안정적으로 학습되었고, **98.44%**의 높은 정확도를 기록했다. |
| 0.00001 | 학습률이 너무 작아 학습 속도가 느렸고 정확도도 낮았다. |

---

6.4 Optimizer 실험 회고

Optimizer 비교에서는 Adam과 SGD의 차이가 뚜렷하게 나타났다.

Adam은 최종 테스트 정확도 **98.45%**를 기록한 반면, SGD는 **92.11%**에 머물렀다. SGD는 단순한 방식으로 파라미터를 업데이트하기 때문에 구현과 이해는 쉽지만, 이번 실험 조건에서는 Adam보다 학습 효율이 낮았다.

Adam은 각 파라미터마다 학습률을 조절하는 방식이 포함되어 있어 더 빠르고 안정적으로 수렴했다. 이를 통해 같은 모델 구조를 사용하더라도 Optimizer 선택에 따라 결과가 크게 달라질 수 있다는 것을 확인했다.

---

6.5 Dropout 실험 회고

Dropout 실험에서는 Dropout 비율에 따라 과적합과 과소적합 양상이 다르게 나타났다.

Dropout을 사용하지 않거나 낮은 비율로 사용한 경우, Train Accuracy가 **100%**까지 올라갔다. 하지만 Test Accuracy는 약 92%대에 머물렀기 때문에 학습 데이터에는 지나치게 잘 맞지만, 테스트 데이터에 대한 일반화 성능은 충분하지 않은 과적합이 발생했다고 볼 수 있다.

Dropout 비율이 0.5일 때는 Train Loss가 너무 낮아지는 것을 어느 정도 막아주었고, Best Test Loss도 가장 낮은 편이었다. 즉, Dropout이 모델의 과적합을 완화하는 데 도움이 되었다.

반면 Dropout 비율이 0.9일 때는 너무 많은 뉴런을 비활성화하여 학습 자체가 어려워졌다. 이 경우 Train Accuracy와 Test Accuracy가 모두 낮아졌기 때문에 과소적합이 발생했다고 볼 수 있다.

따라서 Dropout은 과적합을 줄이는 데 도움이 되지만, 너무 크게 설정하면 모델이 충분히 학습하지 못한다는 점을 확인할 수 있었다.

---

6.6 전체 회고

이번 실험을 통해 신경망의 성능은 단순히 모델 구조만으로 결정되는 것이 아니라, batch size, learning rate, optimizer, dropout 같은 학습 설정에 크게 영향을 받는다는 것을 알 수 있었다.

특히 다음과 같은 점을 배웠다.

* Batch Size는 학습 시간과 성능 사이의 균형을 결정한다.
* Learning Rate는 너무 크거나 작으면 학습이 제대로 되지 않는다.
* Adam Optimizer는 SGD보다 더 빠르고 안정적으로 수렴했다.
* Dropout은 과적합을 줄이는 데 효과적이지만, 비율이 너무 크면 과소적합이 발생한다.
* Train Loss만 낮다고 좋은 모델은 아니며, Test Loss와 Test Accuracy를 함께 확인해야 한다.
