## 1. 실험 목적

본 실험의 목적은 PyTorch, TensorFlow 같은 딥러닝 프레임워크 없이 NumPy만 사용하여 MNIST 손글씨 숫자 분류 신경망을 직접 구현하고, 모델 구조와 학습 조건 변화가 테스트 정확도에 어떤 영향을 주는지 확인하는 것이다.

구현 대상은 ReLU, Softmax, Affine, BatchNorm, Dropout, Cross Entropy Loss, SGD/Adam Optimizer, 학습 루프이며, 최종 목표는 테스트 정확도 97% 이상을 달성하는 것이다.

---

## 2. 모델 구조

| 구분 | 내용 |
| --- | --- |
| **입력** | 784차원 벡터, 28x28 MNIST 이미지를 0~1 범위로 정규화 |
| **은닉층** | Affine -> BatchNorm -> ReLU -> Dropout 순서로 구성 |
| **출력층** | Affine(10) -> Softmax |
| **가중치 초기화** | He 초기화, bias는 0으로 초기화 |

기본 모델 구조는 다음과 같다.

```text
784 -> Affine(512) -> BatchNorm -> ReLU -> Dropout
    -> Affine(256) -> BatchNorm -> ReLU -> Dropout
    -> Affine(10) -> Softmax
```

최종 실험에서는 은닉층 크기를 키워 다음 구조도 비교했다.

```text
784 -> Affine(1024) -> BatchNorm -> ReLU -> Dropout
    -> Affine(512) -> BatchNorm -> ReLU -> Dropout
    -> Affine(10) -> Softmax
```

## 3. 학습 설정

최종 최고 성능 실험의 학습 설정은 다음과 같다.

| 항목 | 값 |

| **hidden_sizes** | [1024, 512] |
| **BatchNorm** | True |

| **Dropout 비율** | 0.3 |

실험에서는 epochs, learning rate, dropout_ratio, hidden_sizes를 단계적으로 변경하며 정확도 변화를 비교했다.

## 5. 결과

### 5.1 실험 결과 요약

| 테스트 | hidden_sizes | BatchNorm | Dropout | dropout_ratio | optimizer | lr | epochs | batch_size | test accuracy | total params |
| --- | --- | --- | --- | --- | --- | --- | --- | --- | --- | --- |
| 테스트 1 | [512, 256] | True | True | 0.5 | Adam | 0.001 | 20 | 128 | 96.35% | 537,354 |
| 테스트 2 | [512, 256] | True | True | 0.5 | Adam | 0.001 | 30 | 128 | 96.88% | 537,354 |
| 테스트 3 | [512, 256] | True | True | 0.5 | Adam | 0.0005 | 30 | 128 | 96.85% | 537,354 |
| 테스트 4 | [512, 256] | True | True | 0.3 | Adam | 0.001 | 30 | 128 | 97.33% | 537,354 |
| 테스트 5 | [1024, 512] | True | True | 0.3 | Adam | 0.001 | 30 | 128 | 97.91% | 1,336,842 |

최종 최고 성능은 테스트 5에서 나왔다.

| 항목 | 값 |
| --- | --- |
| **테스트 정확도** | 97.91% |
| **총 파라미터 수** | 1,336,842 |

### 5.2 실험별 해석

테스트 1과 테스트 2는 모델 구조, optimizer, learning rate, batch size, dropout 비율이 모두 동일하고 epochs만 20에서 30으로 증가시킨 실험이다. epochs를 늘리면 전체 데이터셋을 더 여러 번 반복 학습하므로 파라미터 업데이트 횟수가 증가한다. 그 결과 테스트 정확도가 96.35%에서 96.88%로 상승했다. 파라미터 수는 모델 구조가 동일하므로 537,354개로 변하지 않았다.

테스트 3은 테스트 2와 같은 조건에서 Adam의 learning rate만 0.001에서 0.0005로 낮춘 실험이다. learning rate가 작아지면 한 번의 업데이트에서 파라미터가 이동하는 폭이 줄어 학습이 더 안정적일 수 있지만, 같은 epoch 수 안에서는 수렴 속도가 느려질 수 있다. 그 결과 테스트 2의 96.88%보다 약간 낮은 96.85%의 정확도를 보였다. 차이가 0.03%로 매우 작기 때문에 초기화, 데이터 셔플, Dropout mask 같은 난수 영향도 함께 고려해야 한다.

테스트 4는 테스트 2와 같은 조건에서 dropout_ratio만 0.5에서 0.3으로 낮춘 실험이다. dropout_ratio=0.5는 학습 중 은닉층 뉴런의 절반을 비활성화하므로 정규화 효과는 크지만 학습에 활용되는 정보가 많이 줄어들 수 있다. 반면 dropout_ratio=0.3은 더 많은 뉴런 출력을 유지하면서도 과적합 방지 효과를 제공한다. 이 실험에서는 0.3이 더 적절한 정규화 강도로 작용하여 정확도가 97.33%로 상승했다.

테스트 5는 테스트 4와 같은 학습 조건에서 hidden_sizes를 [512, 256]에서 [1024, 512]로 확장한 실험이다. 네트워크 구조가 784 -> 512 -> 256 -> 10에서 784 -> 1024 -> 512 -> 10으로 변경되었고, 총 파라미터 수는 537,354개에서 1,336,842개로 증가했다. 파라미터 수가 증가하면서 모델의 표현력이 커져 MNIST 숫자의 다양한 획, 기울기, 위치 차이를 더 잘 학습할 수 있었고, 테스트 정확도가 97.91%로 상승했다. 모델이 커질수록 과적합 위험도 증가하지만 BatchNorm과 Dropout을 함께 사용하여 학습 안정성과 정규화 효과를 유지했다.

### 5.3 손실 커브 요약

각 실험에서 epoch가 진행될수록 loss가 감소하는 흐름을 보였다. 특히 epochs를 20에서 30으로 늘렸을 때 추가 학습을 통해 정확도가 개선되었고, dropout_ratio를 0.3으로 낮춘 뒤 모델 크기를 확장했을 때 가장 높은 정확도를 얻었다.

## 6. 회고

이번 실험을 통해 정확도 향상에는 단일 요소보다 여러 조건의 조합이 중요하다는 것을 확인했다. epochs를 늘리면 학습 반복이 증가하여 성능이 개선될 수 있었고, learning rate를 낮추면 업데이트 폭이 줄어 더 안정적일 수 있으나 같은 epoch 안에서는 수렴이 느려질 수 있었다.

Dropout 비율은 성능에 큰 영향을 주었다. dropout_ratio=0.5는 정규화가 강해 학습 정보가 많이 줄어드는 경향이 있었고, dropout_ratio=0.3은 정규화와 정보 유지 사이에서 더 좋은 균형을 보였다.

가장 큰 성능 향상은 hidden_sizes를 [1024, 512]로 확장했을 때 나타났다. 파라미터 수가 증가하면서 모델 표현력이 커졌고, BatchNorm과 Dropout을 함께 적용한 덕분에 큰 모델에서도 비교적 안정적으로 학습할 수 있었다.

최종적으로 [1024, 512] 은닉층, BatchNorm=True, Dropout=True, dropout_ratio=0.3, Adam(lr=0.001), epochs=30, batch_size=128 조건에서 테스트 정확도 97.91%를 달성하여 목표 정확도 97% 이상을 만족했다.
